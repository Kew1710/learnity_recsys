# Learnity — Integration Contract

Learnity — адаптивная система рекомендаций заданий и отслеживания прогресса.
Банк задач и база учеников живут на стороне бекэнда. Learnity отвечает за:
- выбор следующего задания для ученика
- обновление mastery после ответа
- управление учебным планом
- сигналы о застревании (teacher alerts)

---

## Входящие данные (бекэнд → Learnity)

### Создание ученика

| Поле | Тип | Описание                                                                       |
|---|---|--------------------------------------------------------------------------------|
| `student_id` | UUID? | UUID ученика из базы бекэнда. Не генерить падать с ошибкой если не передали id |
| `grade` | int (5–11) | Класс ученика. Используется для cold-start и фильтрации задач                  |


### Регистрация ответа

| Поле | Тип           | Описание                                                               |
|---|---------------|------------------------------------------------------------------------|
| `task_id` | UUID          | На какое задание                                                       |
| `score` | float 0.0–1.0 | Правильность ответа: 1.0 верно, 0.0 неверно,                           |
| `hints_used` | int           | Сколько подсказок использовал                                          |
| `primary_kcs` | list[str]     | Темы (KC), которые проверяет эта часть задания                         |
| `secondary_kcs` | list[str] ?   | Второстепенные темы задания                                            |
| `irt_difficulty` | int           | Сложность задания по IRT [0–5]                                         |
*| `recommendation_source` | str?          | Источник рекомендации (передаётся обратно из ответа на запрос задания) |

### Запрос следующего задания ***

| Поле | Тип | Описание |
|---|---|---|
| `student_id` | UUID | Кому рекомендуем |
| `last_subject` | str? | Тема последнего задания (чтобы не давать подряд одну тему) |

## Исходящие данные (Learnity → бекэнд)

### Рекомендация задания

| Поле | Тип           | Описание                                                               |
|---|---------------|------------------------------------------------------------------------|
| `task_id` | str/list[str] | Какое задание выдать. Бекэнд достаёт его из своего банка               |
| `recommendation_source` | str           | Почему выбрано задание + задание. Передаётся обратно при submit_answer |

### Mastery профиль ученика

| Поле                    | Тип           | Описание                                                                     |
|-------------------------|---------------|------------------------------------------------------------------------------|
| "student_id"              | str           | студент                                                                      |
| `kc_id`                 | str           | Тема                                                                         |
| `probability`           | float 0.0–1.0 | Текущая вероятность освоения                                                 |
| `probability_effective` | float 0.0–1.0 | Вероятность с учётом затухания (decay) — снижается если давно не практиковал |
| `last_practiced`        | datetime      | Когда последний раз практиковал тему                                         |
| `attempts_count`        | int           | Сколько раз пробовал задания по этой теме                                    |

### Учебный план

**Создание плана:**

| Поле | Тип | Описание |
|---|---|---|
| `student_id` | UUID | Для кого создаётся план |
| `mode` | str | `target_mastery` — довести до порога по конкретной теме; `coverage` — охватить список тем |
| `target_kc_id` | str? | Целевая тема (только для `target_mastery`) |
| `mastery_threshold` | float | Порог освоения для продвижения шага. По умолчанию 0.80 |

**Ответ:**

| Поле | Тип | Описание                                                               |
|---|---|------------------------------------------------------------------------|
| `plan_id` | str | Идентификатор созданного плана                                         |
| `steps_count` | int | Количество шагов в плане сохраняю в бд кидаю только id                 |
|---|---| ---                                                                    |
| `kc_id` | str | Тема шага                                                              |
| `status` | str | `pending` — не начат, `in_progress` — активный, `completed` — завершён |
| `difficulty_mode` | str | Режим сложности: `zpd`, `consolidate`, `test`                          |


### Teacher alerts

| Поле | Тип | Описание |
|---|---|---|
| `alert_type` | str | `plateau` — mastery не растёт; `replan_requested` — план неэффективен |
| `kc_id` | str | На какой теме застрял ученик |
| `mastery_at_alert` | float | Уровень mastery в момент создания алерта |
| `tasks_spent` | int | Сколько заданий потрачено без прогресса |
| `message` | str | Описание проблемы |
| `created_at` | datetime | Когда создан алерт |
| `resolved_at` | datetime? | Когда решён. `null` если ещё активен |
